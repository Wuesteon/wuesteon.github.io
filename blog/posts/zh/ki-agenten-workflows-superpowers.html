Superpowers：AI 智能体如何学会像真正的团队那样协作

NW Nils Weiser •2025 年 12 月 12 日 •—

一个让 Claude Code 掌握可复用技能、并编排多阶段开发工作流的框架。

如果一个 AI 编码智能体不只是写代码，而是像一位经验丰富的开发者那样工作，会怎么样？带着记录成文的最佳实践、一致的工作流，以及从过往项目中学习的能力？

Superpowers 框架正是要实现这一点：一个教会 AI 智能体掌握可复用技能的系统。

核心理念

"技能赋予你的智能体超能力。它们是记录成文的最佳实践，Claude 会学会并始终如一地加以运用。"

它的独特之处在于：有一个专门用来创建新技能的技能（writing-skills）。当你找到一个想要复用的解决方案时，就可以把它变成一个新技能。

工作流

PHASE 1 头脑风暴

Claude 分析任务、提出问题，并通过苏格拉底式对话打磨想法。不改动任何代码——只是思考，并与用户一起验证。

PHASE 2 Git Worktree 与计划

Claude 在一个新分支上创建隔离的 git worktree，运行项目初始化，并制定一份详细的实施计划，其中每个任务只需 2 到 5 分钟。

PHASE 3 子智能体开发

全新的子智能体逐一处理每个任务。每一项都要经过两个阶段的评审：先看是否符合规格，再看代码质量。

或者：用 executing-plans 进行批量执行，并在其中设置人工检查点。

PHASE 4 收尾分支

验证测试、给出选项（合并 / PR / 保留 / 丢弃），并清理 worktree。

子智能体与代码评审

最强大的特性之一：Claude 可以派出子智能体自主工作，同时主智能体把控全局。工作完成后，这些改动会接受评审。

这个模型提供了多种选择：

  • 监督模式：你观察每一步
  • 自主模式：子智能体独立工作，你评审最终结果
  • 混合模式：视任务复杂度而定的组合方式

测试驱动开发

RED-GREEN-REFACTOR

Superpowers 始终贯彻 TDD：写测试、让它失败、写代码、让它通过、提交。没有测试，就不提交。

test-driven-development 技能不是一条建议，而是工作流中的硬性要求。这正是 Superpowers 与其他框架拉开差距的地方。

技能的创建

这个框架的独特之处在于：它自带一个用来创建新技能的技能（writing-skills）。当你解决了一个问题、想要复用这个方案时，可以直接由它生成一个新技能。

技能就是带有 YAML frontmatter 的简单 Markdown 文件。它们可以包含说明、示例、清单，甚至脚本——凡是 Claude 一致地执行任务所需要的一切。

社区共享

Superpowers 不只是一个本地系统。技能可以通过 GitHub 共享，从而支持协作式开发： 

# Register marketplace
/plugin marketplace add obra/superpowers-marketplace

# Install plugin
/plugin install superpowers@superpowers-marketplace

想象一下：一位性能优化专家把自己的技能分享出来。任何人只要安装，就能立刻用上他的最佳实践——而无需自己再走一遍那条学习曲线。

为什么这很重要

Superpowers 揭示了 AI 智能体开发的走向：

  1. 知识被沉淀成文：最佳实践不再只装在人的脑子里，而是变成可携带的技能
  2. 团队的扩张方式变了：配备了好技能的初级开发者，也能像资深工程师那样工作
  3. 一致性得到保证：再也没有"我是换了种做法"这种时刻
  4. 学习被加速：犯一次错、做一个技能，然后再也不重蹈覆辙
