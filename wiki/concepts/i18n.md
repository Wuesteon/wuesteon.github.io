# Bilingual (DE/EN) duplicated-tree i18n

## Summary

Homepage strings swap client-side via `data-i18n`; the blog is mirrored as separate DE/EN file
trees with URL redirects.

## Details

Two complementary mechanisms:

1. **For shared/homepage UI**: [[i18n-system]] (`js/translations.js`) holds a DE+EN dictionary and
   `setLanguage()` rewrites `[data-i18n]` text in place, persisting to `localStorage` and flipping
   `<html lang>`.
2. **For long-form content**: translation is *structural* — `blog/` + `blog/posts/de/` vs
   `blog/en/` + `blog/posts/en/`, each post a separately authored file (see [[blog-system]]).
   Toggling language on a blog URL **redirects** to the mirror path, with a `slugMap` covering the
   one post whose filename differs (`mit-ai-halluzinationen` ↔ `mit-ai-hallucinations`).

German is the default/canonical language (`lang="de"`, the homepage canonical is the German
bilingual URL). EN is detected from the URL for blog pages and from `localStorage` elsewhere.
Hreflang tags and the sitemap encode the de/en/x-default relationships for SEO (see
[[seo-geo-layer]]).

This structural approach is a direct consequence of the [[static-no-build]] constraint — there is
no data source to render two language variants from, so the files are physically duplicated.
